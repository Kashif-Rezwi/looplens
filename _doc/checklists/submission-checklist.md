# TestSprite Hackathon Submission Checklist

## Purpose

This checklist tracks what LoopLens needs before the TestSprite Hackathon Season 3 submission.

## Context

The submission needs a public repo, live public URL, TestSprite CLI usage, `LOOP.md`, README, and Discord post. A demo video is optional but encouraged.

## Required Items

- [ ] Public GitHub repository
- [ ] Live public app URL
- [ ] TestSprite CLI usage documented
- [ ] `LOOP.md` written by the agent
- [ ] README explains the app
- [ ] README includes live URL
- [ ] README explains what the loop covered
- [ ] Discord post in `#hackathon-submissions`
- [ ] Discord post includes live URL
- [ ] Discord post includes repo link
- [ ] Discord post includes TestSprite account

## Strongly Recommended Items

- [ ] Public LoopLens report for LoopLens itself
- [ ] GitHub Actions integration
- [ ] TestSprite CI evidence
- [ ] Short demo video
- [ ] Build-in-public social post
- [ ] Screenshots for README
- [ ] Final judge summary
- [ ] Final portfolio summary

## Final Review Questions

- [ ] Can a judge understand LoopLens in under 30 seconds?
- [ ] Can a judge see what was tested?
- [ ] Can a judge see what failed and was fixed?
- [ ] Does the public report work without signing in?
- [ ] Does the app work on mobile?
- [ ] Does the repo show a real development loop?
- [ ] Does the README make the hackathon fit obvious?
