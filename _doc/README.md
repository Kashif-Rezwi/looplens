# LoopLens Documentation

## Purpose

This directory contains the working product and development documents for LoopLens. It is meant for humans and AI agents who need to understand what is being built, why it matters, and how to keep the MVP focused.

## How To Use These Docs

Read these documents before changing product scope, architecture, test strategy, or hackathon submission material.

Recommended order:

1. `product-idea.md`
2. `mvp-development.md`
3. `architecture.md`
4. `user-flow.md`
5. `testing-strategy.md`
6. `decisions.md`
7. `checklists/development-checklist.md`
8. `checklists/submission-checklist.md`

## Scope

These docs are planning artifacts. They should guide implementation but should not be treated as application code.

When the product direction changes, update the relevant doc and add a note to `decisions.md`.

## Document Index

- `product-idea.md`: Product problem, users, use cases, positioning, and risks.
- `mvp-development.md`: MVP scope, development phases, data model, and definition of done.
- `architecture.md`: Planned technical architecture and module boundaries.
- `user-flow.md`: Primary user journey and expected screen behavior.
- `testing-strategy.md`: TestSprite, manual, and automated verification plan.
- `decisions.md`: Product and engineering decisions made during development.
- `checklists/development-checklist.md`: Build-phase checklist.
- `checklists/submission-checklist.md`: Hackathon readiness checklist.
- `research/`: Notes from external research or hackathon references.
