# Source Directory

## Purpose

This directory will contain the LoopLens application source once implementation begins.

## Current Scope

No application code exists yet. The folders below are placeholders for future organization only.

## Planned Organization

- `app/`: Future routes, layouts, and page-level composition.
- `components/`: Future shared UI components.
- `features/`: Future feature modules such as importer, timeline, reports, and publishing.
- `lib/`: Future pure utilities such as parsing, scoring, summaries, and exports.
- `server/`: Future server-side persistence, API, or actions.
- `styles/`: Future global styles and design tokens.
- `types/`: Future shared TypeScript types.
- `data/`: Future sample data and fixtures used by the app.

## Usage Rule

When implementation starts, keep business logic in `lib/` or feature modules instead of burying it inside page components.
